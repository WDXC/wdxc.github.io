死锁的避免

实验内容

实验目标

  进程在执行过程中提出资源请求，使用银行家算法决定是否进行分配

实验要求

  1. 设置资源总数
  2. 新建进程(进程名称、最大需求数)
  3. 提出资源请求 (进程名称、需要资源数)
  4. 资源分配情况显示
  5. 结束

实验代码

银行家算法介绍

  银行家算法 : 实质就是保证系统动态分配资源后不进入不安全状态，以避免可能产生的死锁。 即没当进程提出进程请求且系统资源能够满足请求时,系统将判断此次资源请求后系统是否安全。如果判断是安全，则分给该进程资源，否则不分配资源，申请资源的进程将被阻塞。
  银行家算法执行有个前提，即要求进程预先提出自己的最大资源请求，并假设系统拥有固定的资源总量。

银行家算法的数据结构

1. 可用资源向量available,它记录系统中各类资源的当前可利用数目
2. 最大需求矩阵 max ,它记录每个进程对各类资源的最大需求量。
3. 分配矩阵 allocation ,它记录每个进程对各类资源当前的占有量。
4. 需求矩阵   need , 它记录每个进程对各类资源需要的数目(即等于最大需求矩阵与分配矩阵的差)。
5. 请求向量 request ,它记录某个进程当前对各类资源的申请量，是银行家算法的入口参数。

算法描述

  设进程Pi 向系统提出requesti资源请求 :

  1. 若 requesti > needi,则进程Pi出错
  2. 若 requesti < available,则进程Pi阻塞
  3. 系统试着把资源分配给进程Pi,并对相应数据结构作如下修改 :
    available - requesti ; allocationi + requesti ; needi - requesti;
  4. 系统执行安全性检测子算法，以判断试分配后系统状态是否安全
  5. 若第4步返回真值，即安全，则完成本次分配，返回
  6. 否则，撤销此次(即第三步中的试分配) ,进程Pi阻塞

算法的组成

银行家算法主要是由 Safety algorithm 和 Resource request algorithm 组成

Safety algorithm

判断系统是否处于安全状态的算法可以描述如下 :

  1. Let Work And Finish be vectors of lengh ‘m’ and ‘n’ respectively
    Initialize: Work = Avaliable
    Finish[i] = false;for = 1,2,3,4…n
  2. Find an i such that both
    a. Finish[i] = false
    b. Needi <= Work
    if no such i exists goto step(4)
  3. Work = Work + Allocation[i]
    Finish[i] = true
    goto step(2)
  4. if Finish[i] = true for all i
    then the system is in a safe state
Safety algorithm Example

Resource-request algorithm

令Requesti为进程Pi的请求数组。Request[j] = k 表示进程Pi想要k个资源类型Rj的实例。当进程Pi提出资源请求时，将采取一下措施。

  1. if Requesti <= Needi
    Goto step (2) ; otherwise, raise an error condition, since the process has exceeded its maximum claim.
  2. if Requesti <= Avaliable
    Goto step (3); otherwise, Pi must wait, since the resources are not available
  3. Have the system pretend to have allocated the requested resources to process Pi by modifying the state as follows
    Avaliable = Avaliable - Requesti
    Allocationi = Allocationi + Requesti
    Needi = Needi - Requesti
