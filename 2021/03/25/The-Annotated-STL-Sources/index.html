STL源码剖析读书笔记-迭代器概念与traits编程技法

前言

  在了解STL空间配置的基础后，对于解密STL算法与容器，我们还差一把强力的密钥。那就是作为连接算法与容器的桥梁的迭代器。

迭代器设计模式

  对于迭代器来说，它本身就是一种抽象的设计概念。现实中并没有直接对应于这个概念的实物。在《Design Patterns》中也有对于iterator模式的定义：提供一种方法，使之能够依序巡访某个容器所含的元素，而又无需暴露该聚合物的内部表达方式。
  那么对于STL的iterator与GOF的iterator有什么区别呢？对此，我们需要先了解一个编程范式的概念：OOP(面向对象编程)、GP(范型编程)。它们的区别通过下图就很容易的就可以搞清楚

  OOP:将methods与datasudbd到一起(就是方法与数据放到一个类中),通过继承的方法，利用虚函数表(virtual table)来实现运行时类型的判定，也称为”动态多态”,由于运行过程中需要根据类型去检索虚函数表，因此效率相对较低。
  GP:泛型编程，也被称为”静态多态”，多种数据类型在同一种算法或者结构上皆可操作，其效率与针对某特定数据类型而设计的算法或者结构相同，具体数据类型在编译期间确定，编译器承担更多，代码执行效率高。在STL中利用GP将methods和datas实现了分而治之。

  C++ STL库的整个实现采用的就是GP，而不是OOP。GOF的设计模式采用的就是继承关系实现的。因此，相对来讲，C++ STL的实现效率会相对较高，而且也更有利于维护。

  在STL编程结构中，迭代器其实也是一种模板类，迭代器在STL中等到了广泛的应用，通过迭代器，容器与算法可以有机的绑定在一起，只要对算法给予不同的迭代器。如：vecotr::iterator、list::iterator,std::find()就能对不同的窗口进行查找，而无需针对某个容器来设计多个版本。

迭代器类型

  value_type:迭代器所指对象的类型，原生指针也是一种迭代器，对于原生指针int * ，int为指针所指对象的类型，也就是所谓的value type.
  different_type:用来表示两个迭代器之间的距离，对于原生指针，STL以C++内建的ptrdiff_t作为原生指针的different_type.
  reference:是指迭代器所指对象的类型的引用，reference_type一般用在迭代器的 * 运算符重载上，如果value_type是T,那么对应的reference_type就是T&;如果value_type是const T,那么对应的reference_type就是const T&.
  reference:就是相应的指针类型，对于指针来说，最常用的功能就是operator * 与operator->两个运算符。

  iterator_categoly:它的作用是标识迭代器的移动特性与可以对迭代器执行的操作。从iterator_category上，可以将迭代器分为:Input_iterator、Output_iterator、Forward_iterator、Bidirectional_iterator、Random Access Iterator五类，这样细分也是为了尽可能的提高效率。

迭代器分类

  Input_Iterator:此迭代器不允许修改所指的对象，是只读。支持 ==、!=、++、 * 、-> 等操作
  Output_Iterator:允许算法在这种迭代器所形成的区间内进行只写操作。支持++、 * 等操作
  Forward Iterator: 允许算法在这种迭代器所形成的区间上进行读写操作，但只能意向移动，每次只能动一步。支持Input Iterator 与 Output_Iterator 的所有操作。
  Bidirectional Iterator: 允许算法在这种迭代器所形成的区间上进行读写操作，可双向移动，每次只能移动一步。支持Forward Iterator 的所有操作，并另外支持 “-“操作。
  Random Access Iterator: 包含指针的所有操作，可进行随机访问，随意移动指定的步数。支持前面四种Iterator的所有操作，并另外支持所有的算术运算符。

SGI STL私房菜——__type_traits

  __type_traits提供了一种机制针对不同的类型属性，在编译时期完成函数派送的决定。它提供了五种方式来判断它们是否是trival;

  默认的值为_false_type,如果class内含指针成员，并且对它进行内存动态配置，那么这个class就需要实现出自己的non-trival-xxx。
